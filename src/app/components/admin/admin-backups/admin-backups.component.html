¿Cómo funciona?

Flujo del proceso de respaldo

  1. 1 Servidor acepta la solicitud y responde 202 Accepted de inmediato
  2. 2 pg_dump ejecuta en un hilo dedicado en segundo plano
  3. 3 Se verifica que el archivo generado existe y tiene contenido
  4. 4 El archivo se sube a Supabase Storage (bucket privado)
  5. 5 El archivo temporal del servidor es eliminado
  6. 6 El registro se actualiza a COMPLETADO con los metadatos
info

Los archivos .dump son formato binario de PostgreSQL. Para restaurar se utiliza pg_restore.
